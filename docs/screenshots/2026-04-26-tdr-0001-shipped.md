## Known follow-up (deferred)

**The resolver still generates flat names.** `src/resolver/index.ts` has `nameOverride: \`color.\${rankStr}\`` etc. at lines 130/151/184/204/232/252. This means if anyone runs `npm run ledger:build`, the resolver re-synthesizes `tokens.json` from extractions and **undoes this migration**. The `migrate-tokens.ts` rename would then need to be re-run.

The architecturally-correct fix is to import the rename logic (`classifyColor`, `indexToShade`, `feedbackRoleOf`, `typographyRoleName`, `spacingName`, `radiusName`) into the resolver and apply it in `makeLedgerToken()` before tokens land in the ledger. After that, `migrate-tokens.ts` becomes a one-shot historical artifact.

**Until then:** do not run `ledger:build`. Add a guard to its CLI? Track this as a follow-up ticket.
